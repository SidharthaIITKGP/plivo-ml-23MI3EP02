Constraint Compliance

Pass: 2,000 optimizer steps, 1,923,240 parameters, CPU-only PyTorch training, only train_corpus.txt for model and tokenizer training, no pretrained weights, and no dependencies beyond PyTorch, NumPy, and the standard library.

The final command remains python evaluate.py --checkpoint ckpt.pt --text_file <any_text_file>.

Final Architecture

The model is a decoder-only pre-LayerNorm Transformer with 4 blocks, 4 attention heads, embedding width 188, learned positional embeddings, GELU MLPs, context length 128, dropout 0.0, and untied input/output embeddings. Width 188 was chosen because vocab 512 with width 192 and untied embeddings exceeds the 2M parameter cap by 1,024 parameters, while width 188 stays safely below the cap.

Tokenizer

The final tokenizer is a deterministic 512-token byte-level BPE trained only on train_corpus.txt. IDs 0..255 remain raw bytes, so arbitrary UTF-8 text has fallback coverage. Decoding concatenates token byte sequences and strict-decodes UTF-8, preserving decode(encode(text)) == text.

Token counts: raw dev 159,225 tokens, BPE dev 79,358 tokens. Raw train 7,318,592 tokens, BPE train 3,873,809 tokens.

Training Recipe

Training used AdamW, beta2 0.95, weight decay 0.1, gradient clipping 1.0, batch size 32, peak LR 1.5e-3, 100 warmup steps, and cosine decay to 1.5e-4. The selected checkpoint is the EMA 0.99 version from a 2,000-step run with the same architecture and tokenizer.

What Failed

Weight tying reduced parameters but hurt bpb at 160 width and also lost in the BPE 4x192 tied run. GPT-style scaled initialization was stable but slower in this short training regime. LR 2e-3 did not diverge, but it scored worse than LR 1.5e-3.

Generalization Argument

The final model does not train on dev_eval.txt and uses byte fallback for unseen hidden-set characters. BPE improved both English and Hindi coverage by increasing effective context per byte, while the final model remains comfortably inside the parameter and step caps.
